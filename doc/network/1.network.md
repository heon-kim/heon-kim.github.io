# 1장 네트워크 시작하기

## 개요

### OSI 7계층, 캡슐화(패킷 전송)

## 1. 네트워크 구성도 살펴보기

### 네트워크의 분류

- 서비스를 받는 입장
  - 예를들어
    - 집에서 인터넷에 접속할 때
    - 회사에서 인터넷에 접속할 때
- 서비스를 제공하는 입장
  - 예를들어
    - 클라우드
    - 데이터 센터
    - 회사 기계실 서버

### 1. 홈 네트워크

- 구성
  - 모뎀(네트워크장비)-[케이블]-공유기-[케이블/무선]-단말(노트북/스마트폰/태블릿/데스크톱)

### 2. 데이터센터 네트워크

- 목표: 안정적이고 빠른 대용량 서비스 제공
  - 안정적
    - 다양한 이중화 기술을 사용
    - 높은 통신량을 수용할 수 있어야한다.
- 결과: 고속 이더넷 기술 사용(10G, 25G, 40G, 100G, 400G)
- 구성
  - 기존
    - 3계층 구성
  - 가상화 기술과 높은 대역폭을 요구하는 애플리케이션과 서비스가 등장하면서
    - 2계층 구성 (스파인 - 리프 구조)
      - 서버간 통신이 늘어나는 최근 트래픽 경향을 지원하기 위해 제안되었다.
      - [ ] 최근에는 일반 서버에 10G Base-T 이더넷 포트가 기본적으로 제공되어 TOR 스위치와 연결되고 리프 스위치인 TOR 스위치는 스파인 스위치와 40G, 100G로 연결되는 추세입니다. 더 높은 대역폭을 제공하기 위해 400G 네트워크도 표준화되어 더 높은 대역폭을 사용하는 데이터 센터에 사용되고 있습니다.

## 3.OSI7계층과 TCP/IP

### 1.OSI 7계층

- 7계층 사용 이유
  - 복잡한 데이터 전송 과정을 나누어 보아 이해하기 쉽게 하기 위해
  - 계층별로 프로토콜을 개발화 네트워크 구성 요소들을 모듈화할 수 있다.
- 2분류
  - 하위계층(1~4계층): 데이터 플로 계층
    - 네트워크 엔지니어
  - 상위계층(5~7계층): 애플리케이션 계층
    - 개발자

### 2.TCP/IP 프로토콜 스택

- 4계층 사용 이유
  - 현실에 쉽게 반영
- 2분류
  - 하위계층(1계층): 네트워크 계층
  - 상위계층(2~4계층) : 애플리케이션 계층

## 4.OSI 7계층별 이행하기

### 1계층

- 피지컬 계층
  - 물리적 (전기신호) 전달
  - 주요장비
    - 네트워크 통신 중재
      - 허브 / 리피터
    - 케이블 본체 구성
      - 케이블 / 커넥터
    - 랜카드와 케이블 연결
      - 트랜시버
    - 네트워크 모니터링, 패킷분석을 위해 전기신호를 다른 장비로 복제
      - 탭
  - 주소의 개념이 없고, 전기 신호가 들어온 포트를 제외하고 모든 포트에 같은 전기 신호를 전송

### 2계층

- 데이터 링크 계층
  - 전기신호 -> 모아서 -> 데이터 형태로 처리
  - 주소 정보를 정의하고 정확한 주소로 통신되도록 도와준다.
    - 출발지와 목적지를 구분하고
    - 주소를 확인하여 검사한 후 데이터 처리
      - 플로 컨트롤
        1. 데이터 전송
        2. 스위치 혼잡 상황 발행, 포즈 프레임 전송
        3. 포즈 프레임 수신 후 대기
    - 여러 통신을 구분할 수 있다.
  - 주요 장비
    - 네트워크 인터페이스 카드 (NIC / 네트워크 카드 / 랜 카드 / 이더넷 카드 / 네트워크 어댑터)
      1.  입력된 전기 신호를 데이터 형태로 만든다.
      2.  데이터에서 목적지 MAC 주소와 출발지 MAC 주소를 확인한다.
      3.  네트워크 인터페이스 카드의 고유 MAC 주소를 확인한다.
      4.  목적지 MAC 주소와 네트워크 인터페이스 카드가 갖고 있는 MAC 주소가 맞으면 처리하고 다르면 폐기한다.
    - 스위치
      - MAC 주소를 보고 통신해야 할 포트를 지정해 내보낸다.
      - 단말이 어떤 MAC 주소, 포트로 연결되어 있는지 주소 습득 과정에서 알 수 있다.
        - 1계층 허브와 다른점
          - 허브는 전기 신호를 전체 포트로 전달해준다.
          - 스위치는 필요한 포트만 포트만 사용한다. (네트워크 효율성 증가)

### 3계층

- 네트워크 계층
  - 논리적인 IP주소 정의

## 2.프로토콜

### 프로토콜의 사용

- 물리적 측면의 프로토콜
  - 데이터 전송 매체 / 신호 규약 / 회선 규격 등
  - 이더넷
- 논리적 측면의 프로토콜
  - 통신
  - TCP/IP

### 자연어와 다른 이유

- 한정된 자원으로 통신을 수행해야 하기 때문
  - 최대한 적은 데이터를 이용해 효율적인 프로토콜을 치밀하게 정의하여 통신

### 애플리케이션 레벨의 프로토콜

- 비트 기반이 아닌 문자 기반 프로토콜이 많이 사용
  - HTTP, SMTP
  - 문자를 이용하여 헤더와 헤더값, 데이터가 전송된다.

### TCP/IP 프로토콜 스택 (프로토콜 아님)

- TCP, IP, UDP, ARP, HTTP, SMTP, FTP
- 총 4개 부분으로 나뉜다.
  - 이더넷 (물리 계층)
  - 네트워크 계층
  - 전송 계층
  - 애플리케이션 계층
